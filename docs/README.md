# Innovation Patterns Documentation Site

Docusaurus 3.9.2 static site that publishes project documentation to `https://code.aws.dev/innovation-patterns-0a90b6/` (CI) or `http://localhost:3001` (local).

## Contents

| File/Directory | Purpose |
|----------------|---------|
| `docusaurus.config.ts` | Site config: URL, navbar, footer, themes, markdown settings |
| `sidebars.ts` | All sidebars use `autogenerated` — no manual entries |
| `package.json` | Dependencies: Docusaurus 3.9.2, React 19, Mermaid theme |
| `src/css/custom.css` | Theme overrides |
| `static/` | Static assets (logo, favicon) |
| `docs/` | All markdown content (see below) |

## Convention-Over-Configuration Documentation Rules

The core principle is **docs appear where they are created, driven by front matter, with minimal configuration**.

### Adding a New Document

1. Create the `.md` file in the appropriate section directory.
2. Add front matter with at least `title`:
   ```yaml
   ---
   title: My New Document
   ---
   ```
3. Optionally add `sidebar_position` to control ordering among siblings.
4. That's it. The autogenerated sidebar picks it up automatically.

### Adding a New Category (Subdirectory)

1. Create the directory under the appropriate section.
2. Add an `index.md` (or `README.md`) with front matter:
   ```yaml
   ---
   title: Overview
   ---
   ```
3. **Do NOT create a `_category_.json`** unless you need to explicitly set the category's position relative to siblings or override the label. Docusaurus derives the category label from the directory name and the intro doc from `index.md`/`README.md` automatically.

### When `_category_.json` IS Needed

- To set a category's `position` among siblings when alphabetical order is wrong.
- To override the auto-derived `label` (e.g., directory is `developer-docs` but label should be `Developer Docs`).
- Top-level sections (`getting-started/`, `patterns/`, etc.) use `_category_.json` because their relative order matters for site navigation.

### Front Matter Conventions

| Field | When to Use |
|-------|-------------|
| `title` | **Always required.** Use `Overview` for category landing pages (`index.md`/`README.md`). |
| `sidebar_position` | To control ordering within a category. Lower numbers sort first. |
| `slug` | Only on the site homepage (`getting-started/index.md` uses `slug: /`). |

### `docs/working/` (Local-Only)

The `working/` directory only appears when it exists on disk (checked via `existsSync()` in both `docusaurus.config.ts` and `sidebars.ts`). It is excluded from production builds. Subdirectories follow an aidocs/aicode workflow structure:

| Subdirectory | Purpose |
|--------------|---------|
| `working/specs/` | Feature specifications (README.md + research.md + optional plan.md) |
| `working/docs/` | Document drafts (README.md + research.md + outline.md + draft.md) |

## Known Quirks

- The `baseUrl` switches between `/innovation-patterns-0a90b6/` (CI) and `/` (local) via `process.env.CI`. Local dev always serves from root.
- `onBrokenLinks: 'warn'` — broken links log warnings but do not fail the build. Check console output during `npm run build`.
- Many section landing pages are stubs with "This page is a stub" placeholder text.

## Commands

| Command | Description |
|---------|-------------|
| `npm start` | Dev server on port 3001 with hot reload |
| `npm run build` | Production build to `build/` |
| `npm run serve` | Serve the production build locally |
| `npm run clear` | Clear Docusaurus cache |
| `npx tsc` | TypeScript type check |

## Integration Points

- **CI/CD**: `baseUrl` is environment-aware via `process.env.CI`.
- **Auto-generated content**: `developer-docs/infra/runbook.md` is generated by `/ipa.compose`; `developer-docs/infra/security-disposition.md` is generated by `/ipa.security`. Do not hand-edit these files.
- **Skills**: `/aicode.docs.user` and `/aicode.docs.dev` skills target this docs site for content generation.
