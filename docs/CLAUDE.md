# docs/

Docusaurus 3.9.2 documentation site. Sidebars are **autogenerated** from directory structure — do not create manual sidebar entries.

- Convention over configuration: new docs render automatically via front matter alone. Do NOT create `_category_.json` files unless explicit ordering between siblings is required. Docusaurus sorts alphabetically by default, which is acceptable for most sections.
- Every `README.md` and `index.md` that serves as a category landing page MUST have front matter `title: Overview` unless a more specific title is clearly needed (e.g., the site homepage `getting-started/index.md` which uses `slug: /`).
- Front matter `sidebar_position` on individual `.md` files is the primary ordering mechanism. Prefer this over `_category_.json` `position` fields.
- The `docs/working/` directory is **git-ignored and never deployed** — it exists only for local HITL authoring. Gated by `existsSync()` in config so Docusaurus renders it locally but CI builds never include it. See `docs/working/CLAUDE.md` for its conventions.
- Port is 3001 (`npm start` runs `docusaurus start --port 3001`).
- Mermaid diagrams are enabled globally (`markdown.mermaid: true`).
- `routeBasePath: '/'` — docs serve from the site root, no `/docs/` prefix in URLs.

See README.md for directory layout, configuration files, and content conventions.
