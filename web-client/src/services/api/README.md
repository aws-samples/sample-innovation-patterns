# services/api/

The RTK Query API layer. Contains the base API definition, auto-generated endpoints, cache tag corrections, custom endpoints, and barrel exports.

## Architecture

```
baseApi.ts          ← foundation: createApi(), auth, error handling
    ↑
generated.ts        ← auto-generated endpoints + hooks + types (DO NOT EDIT)
    ↑
enhancedApi.ts      ← cache tag corrections for generated endpoints
    
baseApi.ts          ← also extended by custom endpoint files:
    ↑
uploadApi.ts        ← FormData file uploads
projectsApi.ts      ← full CRUD example with all tag patterns
    
index.ts            ← barrel exports for clean imports
```

## Files

### baseApi.ts

The RTK Query foundation created with `createApi()`. Key aspects:

- `reducerPath: 'appApi'` — must match the key used in the Redux store reducer
- `endpoints: () => ({})` — intentionally empty; endpoints are injected by `generated.ts` and custom API files at import time
- `tagTypes` — lists cache tag categories used by custom endpoints; the codegen adds its own via `enhanceEndpoints({ addTagTypes })`
- `prepareHeaders` — injects an auth token from `localStorage` on every request (replace with the actual auth provider)
- `globalErrorHandler` — module-level setter that allows React components to register error handlers without circular imports

### generated.ts

> [!CAUTION]
> This file is auto-generated by `npm run generate:api`. Do not edit it manually.

Produced by `@rtk-query/codegen-openapi` from `openapi-schema.json`. Contains:

- 6 endpoints (5 Titanic passenger CRUD + health check)
- React hooks for each endpoint (e.g., `useListPassengersApiV1PassengersGetQuery`)
- TypeScript types for all request arguments and response shapes (`TitanicPassengerResponse`, `TitanicPassengerCreate`, etc.)
- Cache tags derived from OpenAPI operation tags

Exports `appApi`, which is the base API enhanced with generated endpoints.

### enhancedApi.ts

Reference example for overriding generated endpoints without editing `generated.ts`.
Use `appApi.injectEndpoints({ overrideExisting: true })` to fix codegen issues like
missing `encodeURIComponent` on path parameters. Currently commented — the passenger
endpoints don't need it because the backend uses `{ticket:path}` in FastAPI.

### uploadApi.ts

Custom endpoint for `FormData` file uploads using `baseApi.injectEndpoints()`. RTK Query auto-detects `FormData` bodies and skips JSON serialization, letting the browser set the correct `multipart/form-data` Content-Type.

### projectsApi.ts

Example full CRUD API demonstrating all cache tag patterns:

| Pattern | Example | Effect |
|---------|---------|--------|
| List-level provide | `providesTags: ['projects']` | Refetches when any `'projects'` tag is invalidated |
| Item-level provide | `providesTags: (r, e, id) => [{ type: 'projects', id }]` | Refetches only for this specific ID |
| List-level invalidate | `invalidatesTags: ['projects']` | Refetches all `'projects'` queries |
| Item + list invalidate | `invalidatesTags: (r, e, { id }) => ['projects', { type: 'projects', id }]` | Refetches lists and the specific item |

Also exports TypeScript interfaces (`ProjectResponse`, `ProjectCreateRequest`) for use in components and hooks.

### index.ts

Barrel exports. Components import from `@/services/api` instead of individual files.

## Adding a Custom Endpoint

1. Create a new file in this directory (e.g., `myFeatureApi.ts`)
2. Import `baseApi` from `./baseApi`
3. Use `baseApi.injectEndpoints()` to define endpoints
4. Export the generated hooks
5. Add any new tag types to the `tagTypes` array in `baseApi.ts`
6. Optionally re-export from `index.ts`
